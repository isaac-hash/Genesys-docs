This guide walks you through installing **ROS 2** and setting up the **Genesys Robotics Framework** on your system.  
Follow each step carefully to ensure a smooth setup experience.

## Prerequisites

<Step>
<StepItem title="System Requirements">

Genesys is built on top of **ROS 2**, so your system must meet the ROS 2 requirements:

- **Operating System**:  
  - Ubuntu 22.04 (recommended)  
  - Ubuntu 24.04 (supported)  
  - Windows & macOS (partial support through containers)
- **Processor**: 64-bit Intel/AMD  
- **RAM**: Minimum 4GB (8GB recommended)  
- **Disk Space**: At least 5GB free

To check your OS version:

```bash
lsb_release -a
````
---

</StepItem>

<StepItem title="Git">
Git is required to clone the Genesys framework repository.

Install Git:

```bash
sudo apt install git -y
```

Verify:

```bash
git --version
```

</StepItem>

<StepItem title="Code Editor">
Recommended editors:

* [Visual Studio Code](https://code.visualstudio.com/)
* [JetBrains Fleet](https://www.jetbrains.com/fleet/)
* Any editor of your choice

</StepItem>

<StepItem title="Required Knowledge">
Before working with Genesys, you should have a basic understanding of:

* **ROS 2 Core Concepts** (nodes, topics, services, actions)
* **Python or C++** (Genesys supports both)
* **Linux terminal usage**
* **Robotics basics** (optional but helpful)

</StepItem>
</Step>

## Installing ROS 2

<Step>

<StepItem title="Setup Sources & Keys">

```bash
sudo apt update && sudo apt install curl gnupg lsb-release
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key -o /usr/share/keyrings/ros-archive-keyring.gpg
```

Add repository:

```bash
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] http://packages.ros.org/ros2/ubuntu \
$(. /etc/os-release && echo $VERSION_CODENAME) main" | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null
```

</StepItem>

<StepItem title="Install ROS 2 Humble (Ubuntu 22.04)">
Recommended for Genesys:

```bash
sudo apt update
sudo apt install ros-humble-desktop-full
```

</StepItem>

<StepItem title="Setup Environment">

Source ROS 2 automatically:

```bash
echo "source /opt/ros/humble/setup.bash" >> ~/.bashrc
source ~/.bashrc
```

Verify installation:

```bash
ros2 --version
```

</StepItem>

</Step>

## Installing Genesys Framework

<Step>

<StepItem title="Clone the Repository">

```bash
git clone https://github.com/genesys-org/genesys.git
cd genesys
```

</StepItem>

<StepItem title="Install Dependencies">

Genesys uses both Python and C++ packages.

### Install Python dependencies:

```bash
pip install -r requirements.txt
```

### Install ROS 2 dependencies:

```bash
rosdep update
rosdep install --from-paths src -y --ignore-src
```

</StepItem>

<StepItem title="Build the Workspace">

Genesys uses a standard ROS 2 colcon workspace layout:

```bash
colcon build --symlink-install
```

Source it:

```bash
source install/setup.bash
```

</StepItem>

<StepItem title="Verify Genesys Installation">

Check available Genesys packages:

```bash
ros2 pkg list | grep genesys
```

Run a demo node:

```bash
ros2 run genesys_core demo_node
```

</StepItem>

</Step>

## Project Structure

The Genesys framework installs the following structure:

```
genesys/
 ├── src/                 # ROS 2 packages (C++ & Python)
 ├── config/              # YAML configuration files
 ├── launch/              # ROS 2 launch files
 ├── worlds/              # Simulation files for Gazebo
 ├── scripts/             # Utility scripts
 ├── docs/                # Documentation
 └── install/             # Built workspace
```

## Running Genesys

<Step>
<StepItem title="Start the Framework">

```bash
source install/setup.bash
ros2 launch genesys_core bringup.launch.py
```

</StepItem>

<StepItem title="Running Simulation">

If you have Gazebo installed:

```bash
ros2 launch genesys_sim simulation.launch.py
```

</StepItem>

<StepItem title="Running Example Robots">

Examples include mobile robot, arm robot, and perception pipelines.

```bash
ros2 launch genesys_examples mobile_robot.launch.py
```

</StepItem>
</Step>
